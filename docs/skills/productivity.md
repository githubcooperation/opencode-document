# 效率类 Skills

效率类 Skill 的核心是“减少重复劳动，把时间留给关键决策”。

## 1. 任务拆解 Skill

- 名称与链接：`task-planner`（示例）
- 用途说明：把大需求拆成可执行的小步骤，并附风险提示。
- 使用示例：

```text
把这个需求拆成 5-7 个可执行步骤，
每步给出验收标准与依赖关系。
```

- 适用场景：需求模糊、任务过大。
- 推荐评级：★★★★★

## 2. 变更摘要 Skill

- 名称与链接：`change-summarizer`（示例）
- 用途说明：自动生成可读的变更说明、影响范围和回滚建议。
- 使用示例：

```text
请根据本次 diff 输出：
- 变更摘要
- 风险点
- 回滚方案
```

- 适用场景：PR 描述、周报、交接文档。
- 推荐评级：★★★★☆

## 3. 会议结论落盘 Skill

- 名称与链接：`meeting-to-actions`（示例）
- 用途说明：把会议记录快速转成执行清单和责任分配。
- 使用示例：

```text
将会议纪要整理为：
- 决策
- 待办
- 负责人
- 截止日期
```

- 适用场景：跨团队协作、项目推进。
- 推荐评级：★★★★☆

## 落地建议

1. 优先自动化“高频低价值”工作
2. 让模板固定而不是每次临场发挥
3. 把产出格式写死，便于团队复用
