# 测试类 Skills

这页给出“能直接提效”的测试类 Skill 方向，优先解决测试覆盖不足和回归成本高的问题。

## 1. TDD 流程 Skill（推荐）

- 名称与链接：可结合 [GSD 指南](/skills/get-shit-done)
- 用途说明：把“先测后写、每步可验证”固化为团队流程。
- 安装方式：按对应 Skill 文档安装。
- 使用示例：

```text
请按 TDD 流程执行：
1) 先补失败测试
2) 再最小实现
3) 再重构
每步输出测试结果
```

- 适用场景：核心规则逻辑、回归频繁模块。
- 推荐评级：★★★★★

## 2. 回归修复 Skill

- 名称与链接：`regression-fixer`（示例）
- 用途说明：根据失败测试和错误日志生成最小修复。
- 使用示例：

```text
请读取这次 CI 失败日志，先定位根因，
只给最小修复并附回归测试建议。
```

- 适用场景：CI 红灯、上线前紧急修复。
- 推荐评级：★★★★☆

## 3. 覆盖率补洞 Skill

- 名称与链接：`coverage-boost`（示例）
- 用途说明：按模块优先级补高价值测试，而不是盲目追 100% 覆盖。
- 使用示例：

```text
请优先补 payment 模块的分支测试，
要求覆盖错误路径与边界输入。
```

- 适用场景：历史项目测试债务高。
- 推荐评级：★★★★☆

## 落地建议

1. 先在一个高风险模块试点
2. 固定测试命令入口（如 `pnpm verify`）
3. PR 中强制附测试结果
