## 本周推荐

### GSD（get-shit-done）

- 项目地址：[gsd-build/get-shit-done](https://github.com/gsd-build/get-shit-done)
- 推荐理由：把“想法 -> 计划 -> 执行 -> 验证”做成可复用工作流，适合中大任务而不是零散问答。
- 上手入口：先看我们重写的 [GSD 快速上手指南](/skills/get-shit-done)，再按你的 runtime 执行安装。

## 推荐标准

我们按以下维度筛选：

1. 可复用性：是否能覆盖高频任务
2. 可维护性：规则和模板是否清晰
3. 可观测性：是否便于审查执行结果
4. 上手成本：新成员是否容易学习

## 安装与使用原则

- 优先选择活跃维护的仓库
- 安装前先审查权限与执行范围
- 先在非生产仓库验证
