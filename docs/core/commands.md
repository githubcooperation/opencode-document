# 内置命令

不同版本的 Opencode 命令可能有差异，本文聚焦“如何使用命令思维协作”。

## 常见命令类别

- 帮助类：查看可用参数与子命令
- 会话类：启动交互、切换模式
- 配置类：查看或更新配置
- 调试类：输出诊断信息

## 基础习惯

先看帮助，再执行：

```bash
opencode --help
opencode <subcommand> --help
```

## 命令使用建议

1. 在新环境先跑帮助命令确认版本能力
2. 把高频命令写进团队文档
3. 对关键流程封装成脚本减少手误

## 诊断思路

命令失败时，不要只看最后一行。建议完整保留：

- 执行命令
- 退出码
- 关键报错上下文

再让 Opencode 基于这些信息分析修复。
