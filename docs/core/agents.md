# Agent 系统

在 Opencode 中，Agent（代理）是执行任务的核心单元。你可以把它理解为“有角色约束和执行能力的 AI 协作者”。

## Plan 与 Build 的差异

常见的两种协作模式：

- Plan：先分析、拆分任务、识别风险
- Build：基于计划执行具体修改与验证

推荐流程是“Plan -> Build -> 审查 -> 再迭代”。

## 一个 Agent 通常会做什么

1. 读取代码上下文
2. 推断当前结构和约束
3. 生成可执行步骤
4. 修改文件并运行命令
5. 汇报结果与风险

## 如何让 Agent 更稳

- 明确限制文件范围
- 明确测试/构建要求
- 明确不可触碰区域
- 要求每步先汇报再继续

## 多 Agent 协作思路

复杂任务可拆成并行子任务，但前提是边界清晰。常见拆法：

- Agent A 负责后端接口
- Agent B 负责前端页面
- Agent C 负责测试与 CI

最终由你统一审查和合并。

## 风险与治理

任何 Agent 都可能出现误判。建议：

- 强制测试
- 关键文件双审
- 高风险改动先出计划

Agent 是“加速器”，不是“责任转移器”。
