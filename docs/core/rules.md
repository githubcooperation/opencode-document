# Rules 规则系统

Rules（规则系统）是团队可控使用 Opencode 的关键。它把“口头约定”变成“机器可执行约束”。

## 为什么要 Rules

没有规则时，输出质量依赖个人临场提示词；有规则后，团队可以获得一致行为。

## 常见规则类型

- 安全规则：禁止执行高风险命令
- 代码规则：必须通过 lint/test 才算完成
- 结构规则：只允许改指定目录
- 输出规则：必须列出变更文件与验证结果

## 编写高质量规则

1. 可验证：能明确判断是否满足
2. 可执行：不要写抽象空话
3. 可维护：规则数量适中，避免冲突

## 示例

```text
- 不得修改 migrations 历史文件
- 每次改动必须运行 pnpm test:unit
- 输出必须包含：改动文件列表、命令结果、剩余风险
```

## 规则冲突处理

当规则冲突时，建议定义优先级：

- 安全规则最高
- 项目规则其次
- 会话临时规则最低

## 落地建议

从 3 到 5 条关键规则开始，不要一开始写太多。先跑通，再迭代。
