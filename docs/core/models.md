# 模型与提供商配置

Opencode 支持多种 Provider（提供商）和模型。你可以根据任务类型做组合。

## 常见选择策略

- 日常编码：速度优先模型
- 架构设计：推理能力更强模型
- 成本敏感：更经济模型或本地模型
- 隐私优先：本地推理（如 Ollama）

## 配置原则

1. 先确保单一 Provider 可稳定调用
2. 再增加备用 Provider 兜底
3. 把密钥放环境变量，不入库

## 任务与模型匹配建议

- 大规模重构：倾向高推理模型
- 小 bug 修复：倾向低延迟模型
- 文档生成：中等能力模型通常足够

## 常见问题

- 响应慢：尝试更轻模型或缩小上下文
- 幻觉多：增加约束，要求引用文件路径
- 成本高：降低上下文长度，拆小任务

## 实践建议

维护一份团队模型策略表：

- 默认模型
- 高风险任务模型
- 紧急场景备用模型
- 成本上限

这样即使人员变动，也能保证输出质量和成本可控。
