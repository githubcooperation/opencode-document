# 配置详解

Opencode 的配置可以分为三层：

- 全局层：你机器上的默认行为
- 项目层：仓库内的团队共享配置
- 会话层：当前任务的临时约束

## 配置优先级

通常是“会话 > 项目 > 全局”。

意味着你在当前任务里明确写的约束，应该覆盖默认设置。

## 推荐配置项

1. 默认模型与备用模型
2. 命令执行白名单/限制
3. 文件改动范围限制
4. 输出风格（是否必须附验证结果）

## 团队实践

- 把项目级约束写入仓库
- 把个人偏好放全局层
- 在 PR 模板里要求附带验证命令

## 配置变更管理

把关键配置当代码管理：

- 变更要有说明
- 有回滚方案
- 跑一次回归检查

## 一条经验

如果你希望 Opencode 稳定可控，最重要的不是“提示词技巧”，而是“明确配置 + 明确流程”。
