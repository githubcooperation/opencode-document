# 配置项速查

这页不是完整 schema，而是团队最常用配置的“决策视图”。

## 配置文件位置

常见层级（以官方文档为准）：

1. 用户级配置
2. 项目级配置（仓库内）
3. 会话级临时设置

通常遵循“就近覆盖”原则：会话 > 项目 > 用户。

## 高优先级配置项

### 1) 模型与 Provider

- 默认模型
- 备用模型
- Provider 凭据（环境变量）

建议：按任务场景定义模型档位，而不是所有任务共用一个模型。

### 2) 权限与工具边界

- 命令执行权限
- 可访问目录
- MCP 工具白名单

建议：先最小权限，后按需放开。

### 3) 搜索与上下文策略

- 代码搜索范围
- 是否包含忽略目录
- 上下文压缩策略

建议：先精确范围再放大，避免噪声上下文。

### 4) 规则与输出模板

- 每步是否必须汇报验证结果
- 是否强制附变更文件列表
- 高风险改动是否必须先计划

建议：把这些写进团队规范，减少口头约定。

## 一个推荐的团队配置思路

1. 项目层：放共享规则和默认策略
2. 用户层：放个人模型偏好
3. 会话层：放临时任务限制

## 配置变更流程

1. 先提案（为什么改）
2. 小范围试运行
3. 观察成本/质量变化
4. 再推广到团队默认
