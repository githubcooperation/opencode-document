# MCP 工具集成

MCP（Model Context Protocol）让 OpenCode 可以接入外部工具与数据源。能力越强，治理越重要。

## MCP 能带来什么

1. 访问外部系统数据（工单、知识库、监控）
2. 调用平台工具（搜索、浏览器、API）
3. 让 Agent 在可控边界内完成更完整任务

## 接入优先级建议

1. 只读型工具（低风险）
2. 可写型但可回滚工具（中风险）
3. 高风险变更工具（需审批）

## 最小可用接入流程

1. 注册 MCP server
2. 在配置中声明可用工具
3. 先跑只读验证
4. 再开放有限写入
5. 加审计日志与告警

## 权限边界模板

- 允许：读取文档、查询状态、拉取指标
- 受限：创建变更、触发部署、批量删除
- 禁止：越权访问、无审批写生产

## 常见问题

### 问题 1：工具很多但命中率低

做法：

- 减少工具数量
- 给工具写明确用途描述
- 让提示词指定优先工具

### 问题 2：误调用高风险工具

做法：

- 把高风险工具放审批门
- 在规则里要求“先说明影响再执行”

### 问题 3：排错困难

做法：

- 记录 tool call 参数和返回
- 为每类工具定义标准错误处理
