# IDE 集成

OpenCode 的核心体验在终端，但可以和 IDE 形成高效协作，而不是二选一。

## 三种常见用法

1. 终端主导 + IDE 编辑
2. IDE 触发 + 终端执行验证
3. 会话共享（团队协作审查）

## 推荐工作台布局

- 左侧：IDE（代码阅读与精修）
- 右侧：OpenCode（计划、批量改动、解释）
- 底部：测试/构建终端

这样你可以把 OpenCode 当“执行与分析引擎”，而不是替代编辑器。

## 哪些任务适合交给 OpenCode

1. 跨文件重构
2. 从错误日志反推修复
3. 批量补测试与文档
4. 把大需求拆成执行计划

## 哪些任务建议 IDE 手动做

1. 像素级 UI 微调
2. 复杂交互细节手感打磨
3. 高风险核心逻辑最终确认

## 协作节奏建议

1. 在 IDE 里选定范围并给出目标
2. 在 OpenCode 中先 plan 再 build
3. 回到 IDE 审查 diff
4. 终端跑测试与回归

## 常见问题

### 问题 1：上下文不一致

现象：OpenCode 理解和 IDE 当前焦点不一致。

做法：在提示中显式给路径、文件和边界。

### 问题 2：改动太散

做法：要求“先列修改计划，再按步骤提交变更”。

### 问题 3：提交说明质量差

做法：要求输出“变更点 + 验证结果 + 风险”。
