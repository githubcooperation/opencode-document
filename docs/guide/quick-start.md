# 快速开始

目标：在现有项目里让 Opencode 完成一次“读取上下文 -> 修改文件 -> 运行验证”的闭环。

## 1. 进入项目

```bash
cd your-project
opencode
```

## 2. 给一个明确任务

推荐用“可验证”的指令：

- 新增一个函数并补测试
- 修复某个明确报错
- 为某段逻辑补注释和类型

示例提示词：

```text
请在 src/utils/date.ts 新增 formatDate 函数，支持 yyyy-mm-dd，
并补充对应单元测试，最后运行测试并总结变更。
```

## 3. 审阅执行过程

重点看三件事：

- 它读了哪些文件
- 它修改了哪些文件
- 它运行了哪些命令

## 4. 人工确认结果

即使结果看起来正确，也建议你做最小审查：

```bash
git diff
pnpm test
pnpm build
```

## 5. 迭代优化提示词

如果结果不理想，不要直接放弃。改成更具体的要求通常就能显著提升质量：

- 明确文件范围
- 明确不要改动的模块
- 明确必须通过的测试

## 实战建议

1. 先小任务，再大任务
2. 每次改动控制在可审查规模
3. 用“先计划后执行”降低返工
