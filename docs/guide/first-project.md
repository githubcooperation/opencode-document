# 第一个项目

这篇内容用“新增 Todo API 接口”作为示例，演示一个完整的小项目协作。

## 场景设定

你有一个 Node.js 服务，想新增两个接口：

- `POST /todos` 创建任务
- `GET /todos` 获取任务列表

同时需要：参数校验、单元测试、README 更新。

## 第一步：让 Opencode 先做计划

示例提示词：

```text
请先阅读当前项目结构，给出实现 Todo API 的分步计划，
包括文件改动点、测试策略和风险。
```

你会得到一份可执行步骤。先检查计划是否合理，再进入执行。

## 第二步：按步骤执行

```text
按计划执行第一步：先定义 todo 的类型和存储层，不要改路由。
完成后运行相关测试并汇报。
```

这种“分段执行”的方式比一次性全做更稳定。

## 第三步：验证与回归

每一步结束后都做验证：

```bash
pnpm test
pnpm lint
pnpm build
```

如果失败，把报错原文贴给 Opencode，让它基于真实错误修复。

## 第四步：整理提交

让 Opencode 帮你总结变更点，再自己确认：

- 接口行为是否与需求一致
- 是否有破坏旧功能的风险
- 文档与测试是否同步更新

## 你会得到什么

完成一次这样的项目后，你会形成可复用模板：

1. 需求拆分
2. 小步执行
3. 快速验证
4. 人工把关

这就是长期稳定使用 Opencode 的关键节奏。
