# TDD 实践

TDD（测试驱动开发）非常适合和 OpenCode 配合，因为它天然有“验收边界”。

## 红-绿-重构怎么和 OpenCode 配合

1. 红（Red）：先写失败测试
2. 绿（Green）：写最小实现让测试通过
3. 重构（Refactor）：优化结构但不改行为

## Step 1：先只写测试

```text
请先补充订单金额格式化的单元测试，覆盖：
- 正常输入
- 空值
- 非法字符
先不要实现业务代码。
```

运行测试，确保先失败。

## Step 2：最小实现

```text
现在只实现让上述测试通过的最小代码，
不要做额外优化。
```

## Step 3：重构与防回归

```text
在不改变行为前提下重构该实现，
并确保测试仍然全部通过。
```

## 适合 TDD 的任务

1. 纯函数和工具层
2. 规则计算与校验逻辑
3. 容易回归的核心模块

## 不太适合的任务

1. 一次性脚本
2. 原型期超高不确定需求

## TDD + OpenCode 提示模板

```text
按 TDD 执行：
1) 先补失败测试
2) 再最小实现
3) 最后重构
每步都运行测试并汇报结果
```
