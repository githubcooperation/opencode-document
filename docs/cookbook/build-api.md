# 用 OpenCode 构建 API

目标：用可复用流程完成一个“可上线”的 API 功能，而不是只生成一段代码。

## 场景

你要在现有服务中新增订单接口：

1. `POST /orders`
2. `GET /orders/:id`
3. 参数校验、错误处理、测试、文档同步

## 第 1 步：先让 OpenCode 只做计划

示例提示：

```text
请先阅读项目结构，只输出实现计划：
- 文件改动点
- 数据模型变化
- 测试策略
- 风险与回滚方案
先不要修改代码。
```

## 第 2 步：分批执行（不要一次全做）

建议拆成 3 批：

1. 数据模型与存储层
2. 路由与业务逻辑
3. 测试与文档

每批结束都要求输出：

- 改了哪些文件
- 跑了什么命令
- 还剩什么风险

## 第 3 步：固定验证命令

```bash
pnpm test
pnpm lint
pnpm build
```

如果你有 API 合约测试，再加：

```bash
pnpm test:contract
```

## 第 4 步：PR 前检查

1. 接口是否满足需求与边界条件
2. 错误码是否一致
3. 文档是否更新
4. 旧功能是否回归通过

## 可直接复用的提示模板

```text
目标：实现订单 API（创建 + 查询）
范围：仅允许修改 src/api、src/services、tests、docs
约束：不得改数据库迁移历史
验收：必须通过 test/lint/build，补充接口文档
输出：每一步都列变更文件、验证结果、剩余风险
```

## 你会得到什么

- 更稳定的生成质量
- 更小的回滚半径
- 更容易审查和交接
